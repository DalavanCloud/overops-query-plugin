The change in rate between an event's rate in the active time span and the error rate during the baseline window to be considered a regression. For example, for an event that used to happen 50 of 5,000 calls into the code (rate = 0.01) in the baseline time window and is now happening 80 out of 4,000 calls (0.2), a supplied value of > 0.19 will mark the event as a regression
